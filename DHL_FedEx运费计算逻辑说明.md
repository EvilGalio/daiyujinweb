# DHL / FedEx 运费计算逻辑说明

> 分析对象：`D重量运费.xlsx`  
> 结论：这个工作簿不是一个完整的一键报价系统，更像是“询价信息 + 体积/重量辅助计算 + DHL/FedEx 分区费率库 + 若干手工试算区”。真正的运费计算核心在 **国家/地区分区、计费重量、重量段费率表、币种换算/加成** 四部分。

---

## 1. 影响运费的主要参数

### 1.1 承运商 / 渠道

表内有两套主费率：

- **DHL**
  - `广诚DHL运费`：DHL 基础费率源表，含 RMB、USD、EUR 三组价格。
  - `区域运费DHL`：按国家/地区展开后的 DHL 查价表。
  - `区域`：另一套 DHL/区域费率表，数值略低，像是原始/旧版/另一渠道价。
  - `2KG内文件`：DHL 2kg 内文件价。

- **FedEx**
  - `广诚FedEx运费`：FedEx 基础费率源表。
  - `区域运费FedEX`：按国家/地区展开后的 FedEx 查价表。

所以同一个国家、同一个重量，DHL 和 FedEx 会因为分区规则和基础费率不同，得到不同运费。

---

### 1.2 目的国家/地区

目的地是查表的第一关键字段。

表中按国家/地区映射到：

- DHL：数字分区 `1区 ~ 9区`。
- FedEx：字母/数字代码，例如 `V / A / Z / X / H / F / M / O / U` 等。

例如：

| 国家/地区 | DHL 分区示例 | FedEx 代码示例 |
|---|---:|---|
| 中国香港特别行政区 | 1区 | V |
| 中国澳门特别行政区 | 1区 | A |
| 韩国 | 2区 | Z |
| 中国台湾地区 | 2区 | X |
| 日本 | 3区 | P |
| 澳大利亚 | 5区 | U |
| 埃及 | 8区 | H |
| 南非 | 9区 / FedEx H | H |

查价时，本质上是：

```text
先用目的国家/地区找到分区/代码，再用分区/代码 + 计费重量找到价格。
```

---

### 1.3 重量

表里涉及几种重量概念：

| 参数 | 来源/含义 | 对运费的影响 |
|---|---|---|
| 单重 | `物流询价2!F列` | 单件或单个产品重量 |
| 数量 | `物流询价2!I列` | 和单重一起决定实际总重量 |
| 单箱重量 | `物流询价2!K列` | 单箱毛重，可能用于快递单箱判断 |
| 箱数 | `物流询价2!L列` | 总毛重 = 单箱重量 × 箱数 |
| 总毛重 | `物流询价2!M列` | 公式为 `K列 * L列` |
| 加包装重量 | FedEx/DHL 试算区 | 大重量段会在实际重量基础上加包装重量 |
| 计费重量 | 需要人工/公式确定 | 最终用来查费率的重量 |

`物流询价2` 中已有公式：

```excel
M2 = K2 * L2        // 总毛重/kg = 单箱重量 × 箱数
R2 = I2 * F2        // 产品净重/总重量 = 数量 × 单重
R13 = SUM(R2:R12)   // 汇总重量
```

---

### 1.4 体积 / 体积重量

`体积公式` 工作表是体积辅助计算表，里面按不同形状计算体积，单位最终基本换算成 m³ / CBM。

典型公式：

```excel
圆柱体体积 = (直径/2 × 直径/2) × 长度 × π × 数量 / 1000000
普通长方体体积 = 长 × 宽 × 高 / 1000000
```

表里还有六角体、圆管、锥台等特殊形状的体积公式。

注意：当前工作簿没有看到把体积自动换算成快递体积重的完整公式，例如常见的：

```text
体积重 = 长 × 宽 × 高 / 5000 或 / 6000
```

但从业务逻辑看，快递报价通常应取：

```text
计费重量 = max(实际重量, 体积重量)
```

这个规则需要在正式系统里补上，否则泡货会低估运费。

---

### 1.5 包装重量

在大重量段试算区里，表格会自动给重量增加包装重量。

FedEx 试算区公式：

```excel
IF(C273<=21.9,"",
  IF(C273<=40,C273+4,
    IF(C273<=70,C273+7.5,
      IF(C273<=300,C273+12.5,
        C273*1.0822
))))
```

含义：

| 实际重量 | 加包装后的计费重量 |
|---:|---:|
| ≤ 21.9kg | 不走这个大货逻辑 |
| 21.9kg ~ 40kg | 实重 + 4kg |
| 40kg ~ 70kg | 实重 + 7.5kg |
| 70kg ~ 300kg | 实重 + 12.5kg |
| > 300kg | 实重 × 1.0822 |

DHL 试算区公式类似，但门槛从 33kg 开始：

```excel
IF(C274<=33,"",
  IF(C274<=70,C274+7.5,
    IF(C274<=300,C274+12.5,
      C274*1.0822
)))
```

含义：

| 实际重量 | 加包装后的计费重量 |
|---:|---:|
| ≤ 33kg | 不走大货逻辑 |
| 33kg ~ 70kg | 实重 + 7.5kg |
| 70kg ~ 300kg | 实重 + 12.5kg |
| > 300kg | 实重 × 1.0822 |

## 2. DHL 运费计算逻辑

### 2.1 DHL 基础费率表

`广诚DHL运费` 是 DHL 基础表。

结构大致是：

| 区域 | 工作表位置 | 说明 |
|---|---|---|
| A:C/K 附近 | RMB 价格 | 人民币基础价 |
| M:W 附近 | USD 价格 | 由 RMB 换算/加成得到 |
| Z:AJ 附近 | EUR 价格 | 由 RMB 换算/加成得到 |

表头说明写着：

```text
5kg以内向上增加1-1.5kg；
5kg-10kg向上取2kg；
10-20kg以内向上取3kg；
20kg-30kg以内，以10-20kg的运费进行计算；
单件重量大于10kg，考虑木箱进行固定费用；
30-70kg以内向上取5-10kg；
70-300kg以内向上取10-15kg。
```

这段说明是业务报价规则，但不是所有规则都已经公式化。

---

### 2.2 DHL 小重量段查表

`区域运费DHL` 的表头是：

```text
国家地区 / 代码 / 0.5 / 1 / 1.5 / ... / 30
```

也就是 0.5kg 到 30kg 按 0.5kg 一个重量档查表。

查价逻辑：

```text
DHL 小货价格 = 区域运费DHL[目的国家/地区所在行, 计费重量所在列]
```

例如：

```excel
区域运费DHL!C2 = 广诚DHL运费!C3
区域运费DHL!D2 = 广诚DHL运费!C4
区域运费DHL!C4 = 广诚DHL运费!D3
```

含义是：

- 香港/澳门为 1 区，所以取 `广诚DHL运费` 的 1区列。
- 韩国/台湾为 2 区，所以取 `广诚DHL运费` 的 2区列。
- 每一个国家行都根据自己的分区，横向引用对应分区的重量价格。

---

### 2.3 DHL 大重量段查表

DHL 大重量段在 `广诚DHL运费` 的底部：

| 重量段 | 1区 | 2区 | 3区 | 4区 | 5区 | 6区 | 7区 | 8区 | 9区 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| 30.1-70 | 15.40 | 18.90 | 21.01 | 24.72 | 47.52 | 50.22 | 42.56 | 44.40 | 70.00 |
| 70.1-300 | 17.60 | 22.68 | 24.83 | 28.84 | 50.16 | 53.01 | 45.60 | 51.80 | 80.00 |
| 300.1-99999 | 19.04 | 25.20 | 27.43 | 29.64 | 52.92 | 55.98 | 47.60 | 52.91 | 82.80 |

这里的价格是 **每 kg 单价**，不是整票价。

大货试算区使用的核心公式是：

```excel
单价 = INDEX(重量段单价表,
             MATCH(重量段, 重量段列, 0),
             MATCH(目的地区, 分区表头, 0))
```

然后：

```excel
EUR 运费 = 单价 × 加包装重量 / 7
USD 运费 = 单价 × 加包装重量 / 6
```

也就是这里有两个汇率假设：

```text
USD 折算除以 6
EUR 折算除以 7
```

---

### 2.4 DHL 币种换算 / 加成

`广诚DHL运费` 里 USD/EUR 价格不是纯查表，而是由 RMB 价格公式换算：

```excel
USD价 = RMB价 × 1.2 / 6.5 × 1.2
EUR价 = RMB价 × 1.2 / 7 × 1.2
```

也可以理解为：

```text
先对人民币价格做 1.2 加成，再按汇率折算，再做一次 1.2 加成。
```

所以影响 DHL 外币报价的参数有：

- RMB 原始费率。
- 加成系数 `1.2`。
- USD 汇率假设 `6.5` 或试算区 `6`。
- EUR 汇率假设 `7`。

这里有一点需要注意：同一工作簿里 DHL 的 USD 换算有两个口径：

- 基础 USD 表：除以 `6.5`。
- 大货试算区：除以 `6`。

正式系统建议统一汇率参数，不要写死在不同公式里。

---

## 3. FedEx 运费计算逻辑

### 3.1 FedEx 基础费率表

`广诚FedEx运费` 是 FedEx 的基础表。

表头结构：

```text
重量 / 美国 / 美国 / 澳门 / 越南 / 蒙古 / 文莱 / 以色列/土耳其 / 南美 / 南非/阿联酋 / 德/法 / 芬/奥/俄 / 加/墨 / 印度 / 日本 / 马来西亚 / 泰国 / 菲律宾 / 印度尼西亚 / 澳洲 / 香港 / 台湾省 / 新加坡 / 韩国
```

下一行是代码：

```text
1 / 2 / A / B / D / E / F / G / H / K / M / N / O / P / Q / R / S / T / U / V / X / Y / Z
```

所以 FedEx 的区域不是 DHL 的 1-9区，而是按照 FedEx 的国家/地区代码列查。

---

### 3.2 FedEx 小重量段查表

`区域运费FedEX` 的表头是：

```text
国家地区 / 代码 / 0.5 / 1 / 1.5 / 2 / 2.5 / 3 / 3.5 / ... / 20.5
```

查价逻辑：

```text
FedEx 小货价格 = 区域运费FedEX[目的国家/地区所在行, 计费重量所在列]
```

`区域运费FedEX` 中的价格引用自 `广诚FedEx运费`：

```excel
区域运费FedEX!C2 = 广诚FedEx运费!U4   // 香港 0.5kg
区域运费FedEX!D2 = 广诚FedEx运费!U5   // 香港 1kg
区域运费FedEX!C3 = 广诚FedEx运费!D4   // 澳门 0.5kg
区域运费FedEX!C4 = 广诚FedEx运费!X4   // 韩国 0.5kg
```

---

### 3.3 FedEx 重货价格

`广诚FedEx运费` 里有一块“重货区价格”：

| 重量段 | 说明 |
|---|---|
| 21.0 - 44.0 | 重货每 kg 单价 |
| 45.0 - 70.0 | 重货每 kg 单价 |
| 71.0 - 99.0 | 重货每 kg 单价 |
| 100.0 - 299.0 | 重货每 kg 单价 |
| 300.0 - 499.0 | 重货每 kg 单价 |
| 500.0 - 999.0 | 重货每 kg 单价 |
| 1,000.0 - 9,999.0 | 重货每 kg 单价 |

FedEx 试算区的查价公式是：

```excel
单价 = INDEX(B$236:Y$242,
             MATCH(重量段, A$236:A$242, 0),
             MATCH(FedEx代码, B$235:Y$235, 0))
```

重量段选择逻辑：

```excel
IF(F<=44,"21-44",
 IF(F<=70,"45-70",
  IF(F<=99,"71-99",
   IF(F<=299,"100-299",
    IF(F<=499,"300-499",
     IF(F<=999,"500-999",
      IF(F<=9999,"1000-999","10000-9999")))))))
```

其中 `F` 是加包装后的计费重量。

然后：

```excel
EUR 运费 = 单价 × 加包装重量 / 7
USD 运费 = 单价 × 加包装重量 / 6
```

---

### 3.4 FedEx 附加调整

`广诚FedEx运费` 里有几列公式在原始列上加固定金额：

```excel
Z列 = H列 + 8 × 重量
AA列 = J列 + 11.2 × 重量
AB列 = K列 + 8 × 重量
AC列 = L列 + 8 × 重量
```

在重货区也有类似固定加价：

```excel
Z列 = H列 + 8
AA列 = J列 + 11.2
AB列 = K列 + 8
AC列 = L列 + 8
```

表内备注也写了：

```text
加了8 / 加了11.2 / 2026-5-12 加8
```

这说明部分 FedEx 国家/区域的价格不是原始价，而是在基础价上增加了固定附加费。

## 4. 文件价：2kg 内文件

`2KG内文件` 是独立文件价表，不按普通包裹价走。

结构：

```text
KG / 1区 / 2区 / ... / 9区
0.5 / 1 / 1.5 / 2
```

逻辑：

```text
如果货物类型 = 文件，且重量 ≤ 2kg：
    运费 = 2KG内文件[重量档, DHL分区]
```

它更像 DHL 文件专用价格，不应和包裹价混用。

## 5. 查表逻辑总结

### 5.1 小货查表

适用于：

- DHL：0.5kg ~ 30kg。
- FedEx：0.5kg ~ 20.5kg。

逻辑：

```text
输入：承运商、目的国家/地区、计费重量

1. 按国家/地区找到对应分区/代码。
2. 按计费重量向上取到表头重量档。
3. 在“国家行 × 重量列”交叉处取价格。
```

Excel 形式可表达为：

```excel
=INDEX(价格区域, MATCH(目的国家, 国家列, 0), MATCH(计费重量档, 重量表头, 0))
```

---

### 5.2 大货查表

适用于：

- DHL：约 30kg 以上。
- FedEx：约 21kg 以上。

逻辑：

```text
输入：承运商、目的国家/地区、实际重量

1. 按国家/地区找到分区/代码。
2. 按承运商规则计算“加包装重量”。
3. 按加包装重量判断重量段。
4. 在重货单价表中查出每 kg 单价。
5. 运费 = 每 kg 单价 × 加包装重量。
6. 如需 USD/EUR，再按表内汇率换算。
```

## 6. 当前表格中需要特别注意的问题

### 6.1 工作簿不是完整自动报价器

`物流询价` 和 `物流询价2` 主要是询价信息录入表，只计算了：

- 总毛重。
- 数量 × 单重。
- 汇总重量。

它没有自动连接到 DHL/FedEx 费率表完成最终报价。

---

### 6.2 有部分公式依赖环境或缓存

部分高重量段表格在当前解析环境里显示 `#NAME?` 或 `#VALUE!`，尤其是使用 `INDEX/MATCH`、图片函数、或表内试算区时。原因可能是：

- 使用了 WPS/新版 Excel 特有函数，例如 `DISPIMG`。
- 部分公式没有在当前环境重新计算。
- 某些单元格引用了空值或文本重量段。

但从公式文本和缓存结果看，核心逻辑仍然可以识别。

---

### 6.3 汇率和加成被写死

表内出现了这些硬编码参数：

| 参数 | 出现场景 |
|---|---|
| 1.2 | DHL 价格加成 |
| 6.5 | DHL USD 基础换算 |
| 7 | EUR 换算 |
| 6 | 大货 USD 试算换算 |
| 8 / 11.2 | FedEx 部分区域附加调整 |
| 4 / 7.5 / 12.5 / 1.0822 | 包装重量规则 |

正式做系统时，建议把这些参数放到单独的配置表，不要散落在公式里。

---

## 7. 建议抽象成系统算法

可以把整个运费计算抽象成下面这套流程：

```text
输入：
- carrier：DHL / FedEx
- cargo_type：文件 / 包裹 / 大货
- destination_country：目的国家/地区
- gross_weight：实际毛重
- dimensions：长宽高或体积
- boxes：箱数
- currency：RMB / USD / EUR

步骤：
1. 计算体积重量。
2. 计费重量 = max(实际毛重, 体积重量)。
3. 如果是文件且 ≤2kg，查“2KG内文件”。
4. 否则按 carrier 选择 DHL 或 FedEx。
5. 用 destination_country 查分区/代码。
6. 判断小货还是大货：
   - DHL：≤30kg 走小货表，>30kg 走大货单价表。
   - FedEx：≤20.5kg 走小货表，>21kg 走重货单价表。
7. 小货：计费重量向上匹配重量档，直接取交叉价格。
8. 大货：按包装重量规则修正重量，再按重量段查每 kg 单价。
9. 运费 = 查得价格或 单价 × 加包装重量。
10. 按币种换算，并叠加固定附加费/加成。
```

---

## 8. 最核心公式模板

### 小货价格

```excel
=INDEX(价格矩阵, MATCH(目的国家, 国家列, 0), MATCH(向上取整重量档, 重量表头, 0))
```

### 大货单价

```excel
=INDEX(重货单价矩阵, MATCH(重量段, 重量段列, 0), MATCH(分区或代码, 分区表头, 0))
```

### 大货运费

```excel
=重货单价 × 加包装重量
```

### DHL 外币价

```excel
USD = RMB × 1.2 / 6.5 × 1.2
EUR = RMB × 1.2 / 7 × 1.2
```

### FedEx 部分区域调整

```excel
调整价 = 原价 + 固定加价 × 重量
```

或重货区：

```excel
调整单价 = 原单价 + 固定加价
```

---

## 9. 一句话总结

这个表的运费不是单纯由重量决定，而是由 **承运商、目的国家/地区、分区/代码、计费重量、重量段、是否文件/包裹/大货、包装重量、币种汇率、加成系数、FedEx 特定区域附加调整** 共同决定。核心计算方式是：**先按国家查分区，再按重量查表；小货取整票价，大货取每 kg 单价后乘加包装重量，再做币种换算和加价。**
