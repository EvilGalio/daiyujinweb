## 1. 系统功能详解

### 1.1 报价计算器

#### 1.1.1 操作流程

1. **上传 3D 文件**：客户拖拽或点击上传 STEP/STP 格式的零件文件（最大 50MB）
2. **自动解析**：系统调用几何引擎解析文件，提取零件的三维尺寸（长×宽×高）和质量，生成高清 3D 预览图
3. **选择参数**：客户在信息确认页面选择：
   - **材质**（如 Aluminum 6061-T6，系统自动按密度算出重量）
   - **公差等级**（IT5 精密磨削至 IT11 粗加工，不同等级对应不同加工成本系数）
   - **表面处理**（阳极氧化、喷砂、抛光等，可多选，费用叠加）
   - **数量**（系统根据数量区间自动匹配阶梯折扣系数）
   - **货币**（USD/EUR/CNY/JPY/GBP 等，按最新汇率换算）
4. **获取估价**：点击计算，系统展示费用构成的明细和总价（做出来之后加上随机数）
5. **正式询盘**（后续有机会的话可拓展）：客户点击"申请正式报价"，系统将当前估价连同 STEP 文件存档为正式询盘记录，工程师审核后出具最终报价

#### 1.1.2 报价计算逻辑

系统采用的估价公式为：

> 单价 = (材料成本 + 基础制造成本) × 公差系数 + 各表面处理费用之和
>
> 总价 = 单价 × 数量 × 数量阶梯系数

其中：

- **材料成本** = 零件重量 × 材料单价。重量由 3D 文件解析出的体积乘以用户所选材质的密度得出
- **基础制造成本** = 根据零件最大尺寸所在区间查表确定，尺寸越大成本越高
- **公差系数** = 精度要求越高，系数越大。例如标准加工（IT8）系数为 1.0，精密磨削（IT5）系数为 2.2
- **表面处理费用** = 所选各工艺固定费用的总和
- **数量阶梯系数** = 批量越大折扣越低，例如 1–9 件系数为 1.0，500–999 件系数为 0.72

#### 1.1.3 输入输出

| 项目               | 说明                                                                                                                                                 |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **输入**     | STEP/STP 格式 3D 文件（单个零件）、材质、公差等级、表面处理选项、订购数量、显示货币                                                                  |
| **输出**     | 零件 3D 预览图、尺寸（mm）、体积（cm³）、重量（kg）、费用明细（材料/制造/公差/表面处理/数量折扣）、单位估价、总价（所选货币）、估价有效期及免责声明 |
| **输出性质** | 标注为 "Estimated Quote"（估价），非正式报价。明确排除税费、运费、关税。正式报价需经工程师审核                                                       |

#### 1.1.4 底层实现原理

- **3D 文件解析**：基于 Open CASCADE Technology（工业级开源几何内核，同为 SolidWorks/FreeCAD 底层引擎），读取 STEP 文件（ISO 10303 国际标准格式）的拓扑结构，计算有向包围盒（OBB）获取精确尺寸，通过体积属性积分算法计算零件体积
- **缩略图生成**：利用几何内核的离屏渲染器在 3840×2880 分辨率下渲染等轴测视图，输出高清 PNG 图片
- **重量计算**：体积 × 材料密度。密度取自系统后台维护的材料数据库
- **报价计算**：所有单价、系数、阶梯参数存储于后端数据库，由管理员在后台统一维护。前端页面只发起计算请求并接收结果，不暴露任何商业定价参数

### 1.2 运费计算器

#### 1.2.1 操作流程

1. **选择目的地**：客户从 247 个支持的国家/地区下拉列表中搜索选择目的国
2. **输入重量**：输入货物重量（kg），最小 0.5kg
3. **选择快递商**：默认同时对比 DHL 和 FedEx，可单选
4. **选择货币**：系统根据目的国自动推荐当地货币，客户可手动切换
5. **获取结果**：点击查询，系统并排展示两家快递的运费、时效和备注

#### 1.2.2 计算逻辑

系统内置完整的 DHL 和 FedEx 区域运费表（覆盖 247 个国家/地区，重量范围 0.5kg–30kg）。查询时执行二维匹配：匹配目的国 → 匹配重量所在区间 → 返回对应运费。支持阶梯价和首重/续重两种计价模式。

货币处理：运费表原始币种为人民币（CNY）。当客户选择其他显示货币时，系统按后台维护的最新汇率进行换算。

#### 1.2.3 输入输出

| 项目           | 说明                                                                                       |
| -------------- | ------------------------------------------------------------------------------------------ |
| **输入** | 目的地国家/地区、重量（kg）、快递商（DHL / FedEx 可多选）、显示货币                        |
| **输出** | 各快递商的运费金额（显示货币）、原始币种金额（供参考）、预估时效、备注（如"含燃油附加费"） |

#### 1.2.4 底层实现原理

- **数据存储**：运费表以标准化格式存储在数据库中，包含快递商、目的国、重量区间、运费金额、货币、时效等字段
- **数据更新**：管理员通过后台界面上传最新版 Excel 运费表，系统自动解析并替换旧数据
- **查询原理**：执行结构化数据库查询 —— 匹配国家 → 查找重量覆盖区间 → 返回对应运费

### 1.3 公差计算器

#### 1.3.1 操作流程

1. **输入基本尺寸**：输入零件的公称尺寸（mm），范围 1–3150mm
2. **输入配合代号**：直接输入配合组合（如 H6/k5），或分别选择孔公差带和轴公差带，也可从常用配合预设中快速选取（如 H7/g6 间隙配合、H7/p6 过盈配合）
3. **获取结果**：系统展示三段信息：
   - **轴（Shaft）**：尺寸上下限、公差等级 IT 值、下偏差 ei、上偏差 es
   - **孔（Bore）**：尺寸上下限、公差等级 IT 值、下偏差 EI、上偏差 ES
   - **配合（Fit）**：配合类型（间隙/过盈/过渡）、最大间隙、最大过盈

#### 1.3.2 计算逻辑

严格遵循 **ISO 286-1:2010** 和 **ISO 286-2:2010** 国际标准：

1. **确定尺寸段**：将基本尺寸匹配到 ISO 286 规定的 21 个尺寸分段
2. **计算标准公差 IT**：使用标准公差公式 IT = k × i，其中 i 为公差单位（与尺寸段几何均值相关），k 为公差等级系数（R5 优先数系）
3. **确定基本偏差**：根据公差带字母确定孔和轴的基本偏差值
4. **计算配合类型**：通过比较孔和轴的上下偏差计算最大间隙和最大过盈，判断配合属于间隙配合、过盈配合还是过渡配合

#### 1.3.3 输入输出

| 项目           | 说明                                                                                                                   |
| -------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **输入** | 基本尺寸（mm）、配合组合（如 H6/k5）、或分别指定孔公差带和轴公差带                                                     |
| **输出** | 轴和孔的尺寸上下限（mm）、公差带 IT 值（μm）、上下偏差 ei/es/EI/ES（μm）、配合类型、最大间隙（μm）、最大过盈（μm） |

#### 1.3.4 底层实现原理

- **公式计算为主**：公差等级 IT5–IT18 采用 ISO 286 标准公式实时计算，不依赖硬编码查表，确保所有尺寸段 × 公差等级的排列组合均被覆盖
- **交叉校验**：在标准规定的 21 个尺寸段节点处，将公式计算结果与标准数值表进行交叉校验，确保计算精度
- **偏差计算**：常用公差带（H6/H7/H8、k5/k6/k7、p6、g6、f7、h6/h7/h9、Js7/Js9 等）的基本偏差值按 ISO 标准公式计算

### 1.4 材质标准查表

#### 1.4.1 操作流程

1. **输入牌号**：在搜索框中输入任意国家的材料牌号，支持完整牌号（EN AW-7075）或片段（7075、AW7075、3.4365）
2. **筛选类别**（可选）：按铝合金、不锈钢、碳钢、钛合金、铜合金、工程塑料等大类筛选
3. **获取结果**：系统展示该材料在 12 个国家/地区标准下的等效牌号，高亮匹配关键词，标注数据来源

#### 1.4.2 数据覆盖

| 大类           | 首批覆盖范围                                        |
| -------------- | --------------------------------------------------- |
| 铝合金（锻造） | 1000–8000 系完整覆盖（80+ 种牌号）                 |
| 铝合金（铸造） | Al-Si / Al-Si-Cu / Al-Mg / Al-Cu 系列（30+ 种牌号） |
| 不锈钢         | 303, 304, 304L, 316, 316L, 321, 410, 420, 430, 440C |
| 碳钢           | AISI 1018, 1045, 1060, 1095                         |

支持的标准体系：ISO / EN / DIN / ANSI-AA / BS / AFNOR / UNE / UNS / JIS / CSA / SIS / GB。

#### 1.4.3 搜索原理

- 系统对搜索关键词执行模糊匹配：优先精确匹配，其次前缀匹配，再次包含匹配
- 自动标准化处理（去除空格、统一大小写、去除常见前缀如 "EN AW-"），确保片段搜索也可命中
- 支持反向查找：输入德国 DIN 牌号 "3.4365" 同样能找到对应的美国 AA 牌号 "7075"

#### 1.4.4 输入输出

| 项目           | 说明                                                                              |
| -------------- | --------------------------------------------------------------------------------- |
| **输入** | 搜索关键词（完整牌号或片段）、可选材料类别筛选                                    |
| **输出** | 材料通用名称、大类、12 个标准下的等效牌号（缺失标准显示"—"）、数据来源及采集日期 |

### 1.5 材料重量计算器

#### 1.5.1 操作流程

1. **选择材质**：从材质下拉列表中选择（如 Aluminum 6061-T6），系统展示该材质的密度信息
2. **选择形状**：从六种标准形状中选择（圆棒、方棒、平板/板材、六角棒、圆管、方管），系统展示该形状的示意图和尺寸参数说明
3. **输入尺寸**：根据所选形状填写对应尺寸参数（如圆棒需直径和长度），支持毫米和英寸单位切换
4. **输入数量**：填写件数（默认 1）
5. **获取结果**：系统实时计算并展示单件体积、单件重量和总重量，支持 kg/lb/g 单位切换

#### 1.5.2 支持形状（参考OnlineMetals）及计算公式

| 形状                    | 输入参数                  | 截面积公式                       |
| ----------------------- | ------------------------- | -------------------------------- |
| 圆棒（Round Bar）       | 直径 D、长度 L            | π × D² ÷ 4                   |
| 方棒（Square Bar）      | 边长 A、长度 L            | A²                              |
| 平板（Flat Bar/Plate）  | 宽 W、厚 T、长 L          | W × T                           |
| 六角棒（Hexagonal Bar） | 对边距 S、长度 L          | √3 ÷ 2 × S²                  |
| 圆管（Round Tube）      | 外径 OD、壁厚 WT、长度 L  | π ÷ 4 × (OD² − (OD−2WT)²) |
| 方管（Square Tube）     | 外边长 A、壁厚 WT、长度 L | A² − (A−2WT)²                |

所有形状归结为统一公式：**重量 = 截面积 × 长度 × 密度 × 数量**。现在先把基础的做了，后续补多种形状只是重复工作。

#### 1.5.3 输入输出

| 项目           | 说明                                                            |
| -------------- | --------------------------------------------------------------- |
| **输入** | 材质、形状、对应尺寸参数（mm 或 inch）、数量、目标重量单位      |
| **输出** | 单件体积（cm³）、单件重量、总重量（kg/lb/g）、所使用材质及密度 |

#### 1.5.4 与报价计算器的关系

重量计算器是报价计算器的轻量化补充。当客户没有完整的 STEP 3D 图纸、只有大致尺寸时，可通过重量计算器快速估算零件重量。两个模块共享同一份材质密度数据库，确保计算结果一致。

## 2. 技术方案

### 2.1 整体架构

系统采用前后端分离架构，前端纯静态页面通过 HTTPS 与后端 API 服务器通信：

```
┌─────────────────────────────────────────┐
│            用户浏览器                      │
│  报价计算 · 运费查询 · 公差查询            │
│  材质查表 · 重量计算                      │
└──────────────┬──────────────────────────┘
               │ HTTPS
               ▼
┌─────────────────────────────────────────┐
│         Cloudflare（CDN / 安全防护）       │
│     DNS 解析 · SSL 证书 · DDoS 防护       │
└──────────────┬──────────────────────────┘
               │
     ┌─────────┴─────────┐
     ▼                   ▼
┌──────────┐    ┌──────────────┐
│ 前端托管  │    │  后端 API 服务器│
│ 静态页面  │    │  Flask + OCC  │
│          │    │  SQLite 数据库 │
└──────────┘    └──────────────┘
```

### 2.2 技术选型

| 层级           | 技术选型                          | 说明                                             |
| -------------- | --------------------------------- | ------------------------------------------------ |
| 前端           | HTML5 + CSS3 + JavaScript（原生） | 纯静态页面，加载快、不依赖前端框架               |
| API 网关与防护 | Cloudflare                        | 免费 DNS + CDN + DDoS 防护 + SSL                 |
| 静态托管       | GitHub Pages                      | 前端页面托管，全球 CDN 加速，零运维成本          |
| 后端框架       | Flask（Python）                   | 轻量级 Web 框架，开发效率高                      |
| 数据库         | SQLite（可迁移至 PostgreSQL）     | 单文件数据库，免安装免配置                       |
| 3D 解析引擎    | Open CASCADE Technology (OCC)     | 工业级开源几何内核，ISO 10303 标准 STEP 文件解析 |

### 2.3 核心引擎：Open CASCADE Technology

OCC 是全球应用最广泛的开源 3D 几何建模内核之一，被 FreeCAD、KiCad、IfcOpenShell 等知名项目采用，也为众多商业 CAD/CAM 软件提供底层几何能力。

在本系统中，OCC 承担三个关键任务：

1. **解析 STEP 文件**：读取符合 ISO 10303 标准的 3D 模型文件，提取几何拓扑结构
2. **尺寸与体积计算**：通过有向包围盒（OBB）算法获取零件的精确三维尺寸，通过体积属性积分计算物理体积
3. **3D 预览图渲染**：在高分辨率（3840×2880）下渲染等轴测视图，供客户确认零件形象

### 2.4 数据管理

所有可配置的商业参数（材料单价、公差系数、表面处理费用、数量阶梯折扣、运费表、汇率）统一存储在服务器端数据库，通过管理员专用后台进行维护。前端客户页面不暴露任何定价参数，仅发起计算请求并接收结果。

每次客户计算操作自动生成后台记录，包含：时间、输入参数完整快照、计算结果。这些记录为后续的客户询盘追踪和业务分析提供数据基础。

### 2.5 系统可移植性

本系统从设计阶段即考虑了可迁移性：

- **数据库无关性**：使用 SQLAlchemy ORM 框架，切换 SQLite → PostgreSQL / MySQL 只需修改一行数据库连接字符串
- **前端零依赖**：纯 HTML/CSS/JS，可在任何静态 Web 服务器上运行
- **后端容器化**：支持 Docker 打包部署，在任何支持 Docker 的环境中一键运行
- **域名解耦**：API 请求地址通过配置文件管理，更换域名/服务器只需改一处
- **第三方零锁定**：核心组件（OCC、Flask、SQLAlchemy）均为开源产品，无商业授权依赖

## 3. 安全与数据管理

### 3.1 数据安全

| 维度         | 措施                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| 传输加密     | 全站 HTTPS（Cloudflare 提供 SSL 证书），所有 API 通信加密                   |
| 商业参数保护 | 材料单价、公差系数、折扣阶梯等定价参数仅存储在服务器端，不可从前端页面获取  |
| 客户文件保护 | 上传的 STEP 文件以随机 UUID 重命名存储，不暴露原始文件名，24 小时后自动清理 |
| 管理员认证   | 参数修改和后台管理需用户名/密码认证                                         |
| 速率限制     | 单 IP 每分钟最多 10 次计算请求，防止恶意刷接口                              |
| 跨域限制     | 后端 API 仅允许本站域名发起请求                                             |

### 3.2 数据资产

系统运行过程中沉淀的数据资产包括：

- **材料数据库**：材质物理参数（密度）、商业参数（单价）、12 国标准等效牌号对照
- **运费数据库**：DHL/FedEx 对全球 247 个国家/地区的区域运费
- **询盘日志**：每次客户查询的参数和结果快照，可用于销售分析和客户跟进
- **ISO 公差引擎**：完整实现 ISO 286 标准计算逻辑，可重复使用

以上数据均为企业自有资产，部署在企业自有服务器上，不受任何第三方平台数据导出限制。
