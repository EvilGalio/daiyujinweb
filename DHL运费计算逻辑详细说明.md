# DHL 运费计算表逻辑说明

> 文件：`A重量运费重制版.xlsx`  
> 工作表：`Sheet1`  
> 用途：根据目的国家/地区、货物重量、包装重量规则、DHL 分区价格表、倍率系数和汇率，快速计算 DHL 运费报价。

---

## 1. 表格整体结构

这张表可以分成三大区域：

| 区域 | 单元格范围 | 作用 |
|---|---:|---|
| 国家/地区 + 0.5–30kg 价格表 | `A1:BJ234` | 上方主查表区。通过国家/地区查 DHL 分区，再通过“国家/地区 + 计费重量”查 0.5–30kg 档位的基础价格。 |
| 30kg 以上阶梯单价表 | `A235:J238` | 大货阶梯价格表。用于 30kg 以上货物，根据“重量阶梯 + 分区”查每 kg 基础单价。 |
| 34kg 以下报价计算区 | `A247:H267` | 下方第一块计算区。用于较小重量货物：货物重量 → 加包装重量 → 查 0.5–30kg 表 → ×1.8 → 折算欧元/美元。 |
| 34kg 以上报价计算区 | `A273:H293` | 下方第二块计算区。用于较大重量货物：货物重量 → 加包装重量 → 查阶梯单价 → ×1.7 → 按计费重量计算总价 → 折算欧元/美元。 |

---

## 2. 影响运费计算的参数

这张表的运费结果主要受以下参数影响。

### 2.1 目的国家/地区

目的国家/地区是查表的第一入口。

- 国家/地区名称位于上方主表 `A2:A234`。
- 每个国家/地区对应一个 DHL 分区代码，位于 `B2:B234`。
- 例如：
  - 日本 → `3` 区
  - 澳大利亚 → `5` 区
  - 美国、加拿大、墨西哥 → `6` 区
  - 德国、英国、法国等欧洲国家 → `7` 区
  - 南非、罗马尼亚等部分国家 → `9` 区

在计算区中，国家/地区通常从外部表引用而来，例如：

```excel
A248 = [1]区域运费FedEX!A247
A274 = [1]区域运费FedEX!A273
```

虽然公式里仍带有 `FedEX` 字样，但当前这张表用于 DHL 运费计算。这里可以理解为：下方报价区的“国家/地区”和“重量”数据来自另一个输入表或原始报价表。

---

### 2.2 DHL 分区

DHL 分区由国家/地区查出。

#### 34kg 以下区域

`B248:B267` 直接返回数字分区：

```excel
B248 = VLOOKUP(A248, A$1:BJ$234, 2, 0)
```

含义：

1. 用 `A248` 的国家/地区名称；
2. 到上方主表 `A1:BJ234` 中查找；
3. 返回第 2 列，也就是 `B列` 的 DHL 分区代码；
4. `0` 表示精确匹配。

输出示例：

```text
日本 → 3
澳大利亚 → 5
美国 → 6
德国 → 7
```

#### 34kg 以上区域

`B274:B293` 返回带“区”字的文本分区：

```excel
B274 = VLOOKUP(A274, A$1:BJ$234, 2, 0) & "区"
```

含义与上面一致，只是把数字分区拼成文本格式，例如：

```text
日本 → 3区
澳大利亚 → 5区
美国 → 6区
德国 → 7区
```

这样做是为了和大货阶梯单价表的表头 `B235:J235` 匹配，因为该区域的表头是 `1区、2区、3区……9区`。

---

### 2.3 原始货物重量

原始货物重量位于计算区的 `C列`。

#### 34kg 以下区域

```excel
C248 = [1]区域运费FedEX!C247
```

#### 34kg 以上区域

```excel
C274 = [1]区域运费FedEX!C273
```

`C列` 是用户或外部表输入的货物重量，后续不会直接拿它查价格，而是先通过规则计算出“加包装重量”或“计费重量”。

---

### 2.4 加包装重量 / 计费重量

`F列` 是整张表中最重要的中间字段。

它表示：在原始货物重量基础上，根据包装规则、计费规则或分段规则调整后的重量。后续查表和计算总价都主要使用 `F列`，而不是直接使用 `C列`。

---

### 2.5 基础价格 / 基础单价

`G列` 是查表得到的基础价格。

但要注意：两个计算区的 `G列` 含义不完全一样。

| 区域 | `G列` 含义 |
|---|---|
| 34kg 以下 | 0.5–30kg 档位对应的“基础总价” |
| 34kg 以上 | 30kg 以上阶梯表对应的“基础单价 / kg” |

---

### 2.6 倍率系数

两个计算区使用不同倍率。

| 区域 | 系数位置 | 系数值 | 用途 |
|---|---:|---:|---|
| 34kg 以下 | `H247` | `1.8` | 基础价格 × 1.8，得到人民币报价 |
| 34kg 以上 | `H273` | `1.7` | 基础单价 × 1.7，得到人民币/kg 报价 |

---

### 2.7 汇率 / 折算除数

表格中有欧元、美元价格列：

| 列 | 字段 | 公式逻辑 |
|---|---|---|
| `D列` | 欧元 | 人民币价格 ÷ 7 |
| `E列` | 美元 | 人民币价格 ÷ 6 |

需要特别注意：  
表中 `D246:E246`、`D272:E272` 附近虽然存在数值，例如 `9.8`、`12.37`、`9.5`、`12.37`，但当前实际公式没有引用这些单元格。当前公式是直接写死 `/7` 和 `/6`。

也就是说，按当前表格实际公式：

```text
欧元折算除数 = 7
美元折算除数 = 6
```

如果公司后续希望汇率可维护，建议把公式从 `/7`、`/6` 改成引用固定汇率单元格，例如 `/D246`、`/E246` 或单独建立“汇率参数区”。

---

## 3. 上方查表区：国家、分区、0.5–30kg 价格

### 3.1 国家/地区与分区表

范围：

```text
A1:B234
```

字段：

| 列 | 字段 | 含义 |
|---|---|---|
| `A列` | 国家地区 | 目的国家/地区名称 |
| `B列` | 代码 | DHL 分区代码，1–9 |

`A2:A234` 共包含 233 个国家/地区记录。

计算区通过 `VLOOKUP` 从这里查分区：

```excel
=VLOOKUP(国家地区, A$1:BJ$234, 2, 0)
```

虽然查找范围写成了 `A$1:BJ$234`，但实际返回的是第 2 列，所以本质上是从 `A:B` 中查分区。

---

### 3.2 0.5–30kg 价格表

范围：

```text
C1:BJ234
```

表头：

```text
C1:BJ1 = 0.5, 1, 1.5, 2, 2.5, ... , 29.5, 30
```

也就是说，0.5–30kg 使用每 `0.5kg` 一个档位的价格。

数据区：

```text
C2:BJ234
```

每一行对应一个国家/地区，每一列对应一个计费重量档位。

例如，日本所在行的价格数据表示：

```text
日本 + 0.5kg → 对应 C列价格
日本 + 1kg   → 对应 D列价格
日本 + 1.5kg → 对应 E列价格
……
日本 + 30kg  → 对应 BJ列价格
```

## 5. 34kg 以下货物计算逻辑

### 5.1 计算区字段

范围：

```text
A247:H267
```

表头：

| 列 | 字段 | 含义 |
|---|---|---|
| `A列` | 国家/地区 | 目的国家/地区 |
| `B列` | 区域 | DHL 分区代码 |
| `C列` | 重量 | 原始货物重量 |
| `D列` | 欧元 | 折算后的欧元价格 |
| `E列` | 美元 | 折算后的美元价格 |
| `F列` | 加包装重量 | 根据重量规则调整后的计费重量 |
| `G列` | 价格 | 从 0.5–30kg 表查出的基础总价 |
| `H列` | 人民币价格 | 基础总价 × 1.8 |

其中 `H247 = 1.8` 是倍率系数。

---

### 5.2 第一步：读取国家/地区

公式示例：

```excel
A248 = [1]区域运费FedEX!A247
```

含义：从外部输入区读取目的国家/地区。

---

### 5.3 第二步：根据国家/地区查 DHL 分区

公式示例：

```excel
B248 = VLOOKUP(A248, A$1:BJ$234, 2, 0)
```

含义：

```text
区域 = 根据国家/地区，在 A2:A234 找到对应行，然后返回 B列分区代码
```

---

### 5.4 第三步：读取原始货物重量

公式示例：

```excel
C248 = [1]区域运费FedEX!C247
```

含义：从外部输入区读取货物重量。

---

### 5.5 第四步：计算加包装重量

公式示例：

```excel
F248 =
IF(C248<=5,
   C248+1,
   IF(C248<=10,
      C248+2,
      IF(C248<=16,
         C248+3,
         IF(C248<=20,
            20,
            IF(C248<=26,
               C248-10+3,
               IF(C248<=33,
                  20,
                  IF(C248<=70,
                     C248+7.5,
                     IF(C248<=300,
                        C248+12.5,
                        C248+22
                     )
                  )
               )
            )
         )
      )
   )
)
```

简化成规则表：

| 原始重量 `C` | 加包装重量 `F` | 说明 |
|---:|---:|---|
| `C ≤ 5` | `C + 1` | 小货加 1kg 包装重量 |
| `5 < C ≤ 10` | `C + 2` | 加 2kg 包装重量 |
| `10 < C ≤ 16` | `C + 3` | 加 3kg 包装重量 |
| `16 < C ≤ 20` | `20` | 统一按 20kg 计 |
| `20 < C ≤ 26` | `C - 10 + 3`，即 `C - 7` | 表内实际公式如此写，结果会小于原始重量 |
| `26 < C ≤ 33` | `20` | 统一按 20kg 计 |
| `33 < C ≤ 70` | `C + 7.5` | 加 7.5kg |
| `70 < C ≤ 300` | `C + 12.5` | 加 12.5kg |
| `C > 300` | `C + 22` | 加 22kg |

注意：虽然这个计算区标题可理解为“34kg 以下”，但公式本身写到了 70kg、300kg 和 300kg 以上的情况。只是从当前数据行看，`A248:H267` 主要用于较小重量报价。

另外，`20 < C ≤ 26` 的公式是 `C248-10+3`，也就是 `C - 7`。这不是常见的“加包装重量”写法，但这是当前表格的实际公式，应按实际公式记录。

---

### 5.6 第五步：用“国家/地区 + 加包装重量”查 0.5–30kg 基础价格

公式示例：

```excel
G248 = INDEX(C$2:BJ$234,
             MATCH(A248, A$2:A$234, 0),
             MATCH(F248, C$1:BJ$1, 0))
```

含义拆解：

#### 1）确定行：国家/地区对应哪一行

```excel
MATCH(A248, A$2:A$234, 0)
```

例如：

```text
A248 = 日本
```

则在 `A2:A234` 中找到“日本”的位置，得到对应行号。

#### 2）确定列：加包装重量对应哪个重量档位

```excel
MATCH(F248, C$1:BJ$1, 0)
```

例如：

```text
F248 = 6
```

则在 `C1:BJ1` 的重量表头中查找 `6kg` 对应的列。

#### 3）返回价格

```excel
INDEX(C$2:BJ$234, 国家行位置, 重量列位置)
```

即返回：

```text
该国家/地区 + 该加包装重量 的基础总价
```

示例：

```text
日本，原始重量 5kg
加包装重量 F = 5 + 1 = 6kg
查 日本 + 6kg 的基础价格 = 462.90
```

---

### 5.7 第六步：基础价格乘以 1.8，得到人民币价格

公式示例：

```excel
H248 = G248 * H$247
```

其中：

```text
H247 = 1.8
```

所以：

```text
人民币价格 = 基础价格 × 1.8
```

示例：

```text
基础价格 G248 = 462.90
人民币价格 H248 = 462.90 × 1.8 = 833.22
```

---

### 5.8 第七步：折算欧元和美元

公式示例：

```excel
D248 = H248 / 7
E248 = H248 / 6
```

含义：

```text
欧元价格 = 人民币价格 ÷ 7
美元价格 = 人民币价格 ÷ 6
```

示例：

```text
人民币价格 H248 = 833.22
欧元价格 D248 = 833.22 ÷ 7 = 119.03
美元价格 E248 = 833.22 ÷ 6 = 138.87
```

---

### 5.9 34kg 以下完整计算链路

可以概括为：

```text
国家/地区
  ↓
VLOOKUP 查 DHL 分区
  ↓
读取原始货物重量 C
  ↓
按 IF 分段规则计算加包装重量 F
  ↓
用 国家/地区 + 加包装重量 到 C2:BJ234 查基础价格 G
  ↓
人民币价格 H = 基础价格 G × 1.8
  ↓
欧元价格 D = 人民币价格 H ÷ 7
美元价格 E = 人民币价格 H ÷ 6
```

公式化表达：

```text
Zone = VLOOKUP(Country, A:B, 2, FALSE)

ChargeWeight = PackagingRule_LessThan34(GrossWeight)

BasePrice = INDEX(PriceTable_0_5_to_30kg,
                  MATCH(Country, CountryList, 0),
                  MATCH(ChargeWeight, WeightHeader_0_5_to_30kg, 0))

RMB = BasePrice × 1.8

EUR = RMB ÷ 7
USD = RMB ÷ 6
```

## 6. 34kg 以上货物计算逻辑

| `B列` | 区域 | DHL 分区文本，例如 `3区`、`7区` |
| `C列` | 重量 | 原始货物重量 |
| `D列` | 欧元 | 折算后的欧元总价 |
| `E列` | 美元 | 折算后的美元总价 |
| `F列` | 加包装重量 | 大货计费重量 |
| `G列` | 价格 | 从 30kg 以上阶梯表查出的基础单价/kg |
| `H列` | 人民币单价 | 基础单价 × 1.7 |

其中 `H273 = 1.7` 是倍率系数。

---

### 6.2 第一步：读取国家/地区

公式示例：

```excel
A274 = [1]区域运费FedEX!A273
```

含义：从外部输入区读取目的国家/地区。

---

### 6.3 第二步：根据国家/地区查 DHL 分区，并拼接“区”

公式示例：

```excel
B274 = VLOOKUP(A274, A$1:BJ$234, 2, 0) & "区"
```

含义：

```text
区域 = 国家/地区对应的数字分区 + "区"
```

例如：

```text
日本 → 3区
德国 → 7区
南非 → 9区
```

拼接“区”的原因是：大货阶梯单价表 `B235:J235` 的表头不是数字 `1,2,3`，而是文本 `1区,2区,3区`。

---

### 6.4 第三步：读取原始货物重量

公式示例：

```excel
C274 = [1]区域运费FedEX!C273
```

---

### 6.5 第四步：计算大货加包装重量 / 计费重量

公式示例：

```excel
F274 =
IF(C274<=33,
   "",
   IF(C274<=70,
      C274+7.5,
      IF(C274<=300,
         C274+12.5,
         C274*1.0822
      )
   )
)
```

规则表：

| 原始重量 `C` | 计费重量 `F` | 说明 |
|---:|---:|---|
| `C ≤ 33` | 空值 `""` | 不适用于 34kg 以上区域 |
| `33 < C ≤ 70` | `C + 7.5` | 加 7.5kg |
| `70 < C ≤ 300` | `C + 12.5` | 加 12.5kg |
| `C > 300` | `C × 1.0822` | 按 1.0822 系数放大重量 |

示例：

```text
34kg → 41.5kg
50kg → 57.5kg
100kg → 112.5kg
320kg → 346.304kg
850kg → 919.87kg
```

---

### 6.6 第五步：判断应该使用哪个大货重量阶梯

公式中的阶梯判断：

```excel
IF(F274<=70,
   "30.1-70",
   IF(F274<=300,
      "70.1-300",
      "300.1-999"
   )
)
```

含义：

| 计费重量 `F` | 匹配阶梯 |
|---:|---|
| `F ≤ 70` | `30.1-70` |
| `70 < F ≤ 300` | `70.1-300` |
| `F > 300` | `300.1-999` |

注意：阶梯是根据 `F列` 的加包装重量/计费重量判断，而不是直接根据 `C列` 的原始重量判断。

---

### 6.7 第六步：用“阶梯 + 分区”查大货基础单价

公式示例：

```excel
G274 =
INDEX(B$236:J$244,
      MATCH(IF(F274<=70,
               "30.1-70",
               IF(F274<=300,
                  "70.1-300",
                  "300.1-999"
               )),
            A$236:A$244,
            0),
      MATCH(B274, B$235:J$235, 0)
)
+ CHOOSE(B248,0.8,0.8,0.8,0.8,2.3,13.7,15.8,11.7,11.7)
- CHOOSE(B248,0.8,0.8,0.8,0.8,2.3,13.7,15.8,11.7,11.7)
```

这个公式可以分成两部分。

#### 第一部分：真正的查价逻辑

```excel
INDEX(B$236:J$244,
      MATCH(重量阶梯, A$236:A$244, 0),
      MATCH(区域, B$235:J$235, 0))
```

含义：

1. 根据 `F列` 判断重量阶梯；
2. 在 `A236:A244` 中找到对应阶梯行；
3. 根据 `B列` 的分区，例如 `3区`，在 `B235:J235` 中找到对应分区列；
4. 返回该阶梯、该分区的基础单价/kg。

示例：

```text
日本，原始重量 34kg
计费重量 F = 34 + 7.5 = 41.5kg
F ≤ 70，所以使用阶梯 30.1-70
日本 = 3区
查 30.1-70 + 3区 的基础单价 = 21.01
```

#### 第二部分：CHOOSE 加减项

公式末尾有一组：

```excel
+ CHOOSE(...)
- CHOOSE(...)
```

因为加上的内容和减去的内容完全相同，所以这部分净影响为 `0`。

也就是说，当前公式实际等价于：

```excel
G274 =
INDEX(B$236:J$244,
      MATCH(IF(F274<=70,"30.1-70",IF(F274<=300,"70.1-300","300.1-999")),
            A$236:A$244,0),
      MATCH(B274,B$235:J$235,0))
```

因此，`G列` 当前实际就是大货阶梯表查出来的基础单价/kg，没有额外加价或减价。

---

### 6.8 第七步：基础单价乘以 1.7，得到人民币/kg 报价

公式示例：

```excel
H274 =
G274 * H$273
+ CHOOSE(B248,0.8,0.8,0.8,0.8,2.3,13.7,15.8,11.7,11.7)
- CHOOSE(B248,0.8,0.8,0.8,0.8,2.3,13.7,15.8,11.7,11.7)
```

同样，后面的 `+CHOOSE(...) - CHOOSE(...)` 净影响为 `0`。

所以实际等价于：

```excel
H274 = G274 * H$273
```

其中：

```text
H273 = 1.7
```

所以：

```text
人民币/kg 报价 = 基础单价/kg × 1.7
```

示例：

```text
日本，34kg：
基础单价 G274 = 21.01
人民币/kg 报价 H274 = 21.01 × 1.7 = 35.717
```

---

### 6.9 第八步：按计费重量计算欧元和美元总价

34kg 以上区域和 34kg 以下区域最大的不同是：

- 34kg 以下：`H列` 已经是人民币总价；
- 34kg 以上：`H列` 是人民币/kg 单价，需要再乘以 `F列` 计费重量。

公式示例：

```excel
D274 = H274 * F274 / 7
E274 = H274 * F274 / 6
```

含义：

```text
欧元总价 = 人民币/kg 报价 × 计费重量 ÷ 7
美元总价 = 人民币/kg 报价 × 计费重量 ÷ 6
```

示例：

```text
日本，34kg：
计费重量 F274 = 41.5
人民币/kg 报价 H274 = 35.717

欧元总价 D274 = 35.717 × 41.5 ÷ 7 = 211.75
美元总价 E274 = 35.717 × 41.5 ÷ 6 = 247.04
```

### 6.10 34kg 以上完整计算链路

可以概括为：

```text
国家/地区
  ↓
VLOOKUP 查 DHL 分区，并拼成“x区”
  ↓
读取原始货物重量 C
  ↓
按大货规则计算加包装重量/计费重量 F
  ↓
根据 F 判断重量阶梯：
    F≤70 → 30.1-70
    F≤300 → 70.1-300
    F>300 → 300.1-999
  ↓
用 重量阶梯 + 分区 到 B236:J238 查基础单价 G
  ↓
人民币/kg 报价 H = 基础单价 G × 1.7
  ↓
人民币总价 = 人民币/kg 报价 H × 计费重量 F
  ↓
欧元总价 D = 人民币总价 ÷ 7
美元总价 E = 人民币总价 ÷ 6
```

公式化表达：

```text
ZoneText = VLOOKUP(Country, A:B, 2, FALSE) & "区"

ChargeWeight = PackagingRule_Above34(GrossWeight)

Tier =
  if ChargeWeight <= 70: "30.1-70"
  else if ChargeWeight <= 300: "70.1-300"
  else: "300.1-999"

BaseUnitPrice = INDEX(HeavyPriceTable,
                      MATCH(Tier, TierList, 0),
                      MATCH(ZoneText, ZoneHeader, 0))

RMBUnitPrice = BaseUnitPrice × 1.7

RMBTotal = RMBUnitPrice × ChargeWeight

EUR = RMBTotal ÷ 7
USD = RMBTotal ÷ 6
```

## 10. 需要特别记录的表格细节和潜在注意点

### 10.1 公式中存在外部工作簿引用

计算区的国家和重量来自外部引用，例如：

```excel
=[1]区域运费FedEX!A247
=[1]区域运费FedEX!C247
```

这说明当前表可能是从另一个报价表复制/联动过来的。若原始外部文件不存在，Excel 可能只保留上次缓存值，或者提示更新链接。

建议后续整理时，把输入字段改成本表内可编辑输入区，减少外部链接依赖。

---

### 10.2 34kg 以下区域的加包装重量规则有特殊写法

尤其是：

```excel
IF(C248<=26,C248-10+3,...)
```

即：

```text
20 < C ≤ 26 时，加包装重量 = C - 7
```

这会导致加包装重量小于原始货物重量。例如：

```text
21kg → 14kg
26kg → 19kg
```

这与通常理解的“加包装重量”不一致。  
如果这是公司内部特殊折算规则，需要保留；如果不是，需要重点复核。

---

### 10.3 34kg 以下区域中，26–33kg 被统一按 20kg 查价

公式：

```excel
IF(C248<=33,20,...)
```

含义：

```text
26 < C ≤ 33 时，F = 20kg
```

因此，例如：

```text
32.5kg → 按 20kg 查 0.5–30kg 表
33kg → 按 20kg 查 0.5–30kg 表
```

表中南非 33kg 示例就是按 `20kg` 查价。

---

### 10.4 34kg 以上区域对 33kg 以下会返回空计费重量

公式：

```excel
IF(C274<=33,"",...)
```

因此该区域理论上只适合 34kg 以上的货物。  
如果把 29.5kg、30kg、33kg 这类数据放在 34kg 以上区域，`F列` 会为空，后续价格、欧元、美元可能出现异常或不具备业务意义。

表中有一行意大利 `29.5kg` 出现在 34kg 以上区域，结果欧元/美元为 `#VALUE!`，原因就是该行重量不适用于这个区域。

---

### 10.5 34kg 以上区域中，`CHOOSE(...)` 加减项当前没有实际影响

公式中出现：

```excel
+CHOOSE(...)
-CHOOSE(...)
```

由于加和减完全相同，所以净影响为 `0`。

这可能是历史公式遗留，也可能是原本准备添加分区附加费但后来抵消掉。  
当前计算逻辑应按“无额外附加费”理解。

---

### 10.6 汇率参数目前是硬编码，不是引用汇率单元格

当前公式：

```excel
欧元 = 人民币 ÷ 7
美元 = 人民币 ÷ 6
```

而不是：

```excel
欧元 = 人民币 ÷ 某个欧元汇率单元格
美元 = 人民币 ÷ 某个美元汇率单元格
```

因此如果汇率变化，只改 `D246:E246` 或 `D272:E272` 附近的数值，不一定会影响计算结果。必须同步修改公式，或把公式改为引用汇率单元格。

## 11. 总结版逻辑

### 11.1 小货 / 34kg 以下

```text
输入：国家/地区、原始重量

1. 根据国家/地区查 DHL 分区。
2. 根据原始重量计算加包装重量。
3. 用“国家/地区 + 加包装重量”到 0.5–30kg 表中查基础总价。
4. 基础总价 × 1.8 = 人民币价格。
5. 人民币价格 ÷ 7 = 欧元价格。
6. 人民币价格 ÷ 6 = 美元价格。
```

### 11.2 大货 / 34kg 以上

```text
输入：国家/地区、原始重量

1. 根据国家/地区查 DHL 分区，并转成“x区”格式。
2. 根据原始重量计算加包装重量/计费重量。
3. 根据计费重量判断阶梯：30.1-70、70.1-300、300.1-999。
4. 用“重量阶梯 + 分区”到大货阶梯表中查基础单价/kg。
5. 基础单价/kg × 1.7 = 人民币/kg 报价。
6. 人民币/kg 报价 × 计费重量 = 人民币总价。
7. 人民币总价 ÷ 7 = 欧元总价。
8. 人民币总价 ÷ 6 = 美元总价。
```

## 12. 可维护性建议

为了让这张 DHL 运费表后续更稳定、易维护，建议：

1. **把汇率做成明确参数单元格**  
   不要在公式里写死 `/7` 和 `/6`。建议建立：
   - 欧元汇率参数
   - 美元汇率参数  
   然后公式统一引用参数单元格。

2. **把 `1.8` 和 `1.7` 标注为倍率参数**  
   当前它们放在 `H247` 和 `H273`，但没有明确文字说明。建议在旁边增加“报价倍率/利润系数/销售系数”等说明。

3. **复核 34kg 以下包装重量公式中的特殊区间**  
   特别是：
   - `20 < C ≤ 26` 使用 `C - 7`
   - `26 < C ≤ 33` 使用 `20kg`  
   如果这是业务规则，应写成说明；如果不是，应修正公式。

4. **清理无效的 `+CHOOSE -CHOOSE` 项**  
   当前对结果没有影响，容易误导后续维护人员。可以删除，或者如果它原本代表附加费，就改成真正可控的附加费参数。

5. **减少外部工作簿引用**  
   当前输入数据来自 `[1]区域运费FedEX`，容易造成链接失效。建议改为本表输入区或 Power Query/结构化引用。

6. **统一命名“DHL/FedEX”**  
   当前文件用于 DHL，但部分引用名仍显示 `FedEX`。建议统一命名，避免使用者误判承运商。
